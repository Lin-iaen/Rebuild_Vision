IBVS 极简视觉追踪系统 V2 - 架构与重构文档
1. 系统核心愿景
本项目旨在通过树莓派与 OV5647 摄像头，实现对微弱功率激光点（红/绿）的工业级稳定追踪。
核心战术： 抛弃花哨的彩色画面，抛弃复杂的算法补丁。用“死磕物理底层”代替“软件算法弥补”。系统将只依赖黑白掩膜（Mask）进行运算，极大降低性能开销。

2. 物理与硬件基准 (Golden Parameters)
严禁在代码中随意修改以下经过实测的物理常数。

硬件防频闪与曝光下限： 10000us 为 50Hz 交流电的魔法抵消周期。但在弱光激光笔场景下，为了保证进光量，不苛求防频闪。

ISP 窃听参数（基准工作点）：

Shutter (曝光): 33239 us (底层测算出的极限进光量)

Gain (模拟增益): 8.0 (确保黑胶带上的弱激光依然可见)

AWB (白平衡): Auto (微弱激光下，交由底层自动适应环境)

光学真理： 目标激光在黑胶带上的 V 值，必须物理上大于白纸环境光的 V 值（Margin > 15），否则视为环境光超标，必须从物理层面遮光。

3. 核心工具链 (复用资产)
在正式编写主循环前，本系统依赖以下调试工具进行现场标定：

data_driven_tuner.py (数据裁决仪): 用于在场地发生变化时，通过高斯模糊和 99% 分位数算法，客观评估当前环境光与激光的 V 值落差，判定物理参数是否“有解”。

01_blind_vision.py (盲人视觉流): 最底层的单通道掩膜推流器，用于验证 HSV 阈值和形态学操作是否能切出完美的实心白斑。

4. 软件架构设计 (分层解耦)
V2 版本代码必须严格遵循“数据流水线”模式，禁止跨层调用：

Layer 1: 硬件感知层 (Sensor Layer)

职责: 仅负责使用锁死的参数 (shutter=33239, gain=8.0, awb=auto) 驱动摄像头，拉取原始图像帧。

原则: 内部使用 read1() 防止管道死锁，容忍开机 3-5 秒的底层测光延迟。

Layer 2: 视觉切割层 (Vision Pipeline)

职责: 接收 BGR 画面 -> 转 HSV -> 应用宽容阈值掩膜 -> 执行 15x15 形态学膨胀（将米粒光斑放大为馒头） -> 输出纯净的单通道二值化黑白图像（Mask）。

原则: 该层严禁输出任何彩色调试画面，从源头切断计算压力。

Layer 3: 逻辑与追踪层 (Tracking Logic)

职责: 对 Mask 寻找轮廓计算质心 (cx, cy)。

核心算法 (借鉴 OpenMV 贪吃蛇逻辑): 建立容量为 N 的历史坐标环形队列。绿激光（追击者）不瞄准红激光（目标）的当前点，而是瞄准历史轨迹中距离大于 tracking_distance 的“尾巴”，从物理上彻底解决光斑重叠导致的颜色畸变问题。

Layer 4: 执行与运动层 (Actuator/Control)

职责: 接收相对误差坐标 (dx, dy)，通过 PID 等控制算法输出舵机/电机的 PWM 信号。

5. 后续重构步骤建议
构建并验证 Layer 1 & 2（输出稳定的质心坐标）。

实现 Layer 3 的历史队列与目标筛选逻辑。

接入云台/底盘驱动，完成闭环。
