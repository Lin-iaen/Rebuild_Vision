IBVS 极简视觉追踪系统 V2 - 架构与重构文档
1. 系统核心愿景
本项目旨在通过树莓派与 OV5647 摄像头，实现对微弱功率激光点（红/绿）的工业级稳定追踪。
核心战术： 抛弃花哨的彩色画面，抛弃复杂的算法补丁。用"死磕物理底层"代替"软件算法弥补"。系统将只依赖黑白掩膜（Mask）进行运算，极大降低性能开销。

2. 物理与硬件基准 (Golden Parameters)
严禁在代码中随意修改以下经过实测的物理常数。

硬件防频闪与曝光下限： 10000us 为 50Hz 交流电的魔法抵消周期。但在弱光激光笔场景下，为了保证进光量，不苛求防频闪。

ISP 窃听参数（基准工作点）：

Shutter (曝光): 33239 us (底层测算出的极限进光量)

Gain (模拟增益): 8.0 (确保黑胶带上的弱激光依然可见)

AWB (白平衡): Auto (微弱激光下，交由底层自动适应环境)

光学真理： 目标激光在黑胶带上的 V 值，必须物理上大于白纸环境光的 V 值（Margin > 15），否则视为环境光超标，必须从物理层面遮光。

3. LAB 色彩空间检测参数
LAB 通道阈值（tracker.py）：

激光中心检测：L > 180, 125 < A < 140
激光光晕检测：L > 60, 130 < A < 145
A 值上界作用：排除边缘色差伪影（A=143-157），避免误检
ROI 掩膜：排除画面边缘 10%，避免边缘干扰

诊断数据参考：
白色背景激光：L=254, A=127
黑色背景激光：L=250, A=132
边缘色差伪影：L=228, A=143-157

4. 核心工具链 (调试与标定工具)
本系统依赖以下调试工具进行现场标定，所有工具均通过 MjpegStream 推流，浏览器可直接查看：

sample/test_camera_stream.py (摄像头直推): 验证摄像头采集是否正常。

sample/test_tracker_stream.py (激光追踪流): 实时展示 LAB 色彩空间激光检测效果。

sample/test_rectangle.py (矩形标定): 检测黑色电工胶带矩形框，标定中心线角点。

sample/test_align.py (云台坐标系标定): 通过发送已知角度指令，记录激光轨迹，反解摄像头与云台坐标系的 2×2 变换矩阵。

sample/test_lab_diagnostic.py (LAB 值诊断): 显示激光点的实际 LAB 值，帮助调整检测阈值。

5. 软件架构设计 (分层解耦)
V2 版本代码必须严格遵循"数据流水线"模式，禁止跨层调用：

Layer 1: 硬件感知层 (Sensor Layer)

职责: 仅负责使用锁死的参数 (shutter=33239, gain=8.0, awb=auto) 驱动摄像头，拉取原始图像帧。

原则: 内部使用 read1() 防止管道死锁，容忍开机 3-5 秒的底层测光延迟。

Layer 2: 视觉切割层 (Vision Pipeline)

职责: 接收 BGR 画面 -> 转 LAB 色彩空间 -> 应用 L/A 通道阈值掩膜（含 A 值上界）-> ROI 掩膜排除边缘 -> 执行 5×5 形态学闭+开操作 -> 输出检测结果。

原则: 使用 LAB 色彩空间（不是 HSV），更稳定。详见 tracker.py。

Layer 3: 逻辑与追踪层 (Tracking Logic)

职责: 对 Mask 寻找轮廓计算质心 (cx, cy)。

控制策略:
- 闭环反馈：摄像头实时检测激光位置，计算误差，发送指令
- 噪声剔除：距离上次位置超过 50px 则判定为噪声，丢弃
- 检测失败：发送零指令，云台保持静止
- 控制频率：20fps（50ms 延时）

Layer 4: 执行与运动层 (Actuator/Control)

职责: 接收相对误差坐标，通过云台协议发送单轴指令。新协议: 0x02 0x01/0x02 <value_i16>（X/Y轴，值 = 角度 × 100）。

标定: 使用 test_align.py 进行坐标系标定，获取 2×2 变换矩阵 M⁻¹，将像素误差转换为云台角度指令。
